Module: GitHub and GitHub Desktop

Introduction to GitHub

GitHub is a platform for version control and collaboration, allowing multiple people to work on projects simultaneously.

It uses Git, a distributed version control system, to track changes and manage code.

Setting Up GitHub

Create a GitHub account by visiting github.com and signing up.

Configure Git with your GitHub credentials using the command:

git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

Installing Git

For Mac:

Download the Git installer from git-scm.com.

Open the downloaded .dmg file and follow the instructions to install Git.

Verify the installation by running git --version in the Terminal.

For Windows:

Download the Git installer from git-scm.com.

Run the downloaded .exe file and follow the installation wizard steps.

Verify the installation by running git --version in Command Prompt or PowerShell.

Introduction to GitHub Desktop

GitHub Desktop provides a graphical interface to interact with GitHub repositories.

Download and install GitHub Desktop:

For Mac:

Download GitHub Desktop from desktop.github.com.

Open the downloaded .dmg file and drag the GitHub Desktop icon into the Applications folder.

Launch GitHub Desktop from the Applications folder and sign in with your GitHub credentials.

For Windows:

Download GitHub Desktop from desktop.github.com.

Run the downloaded .exe file and follow the installation steps.

Launch GitHub Desktop from the Start menu and sign in with your GitHub credentials.

Creating a Repository

On GitHub, click the “+” icon and select “New repository.”

Name your repository, add a description, and choose the visibility (public or private).

Initialize with a README if desired, then click “Create repository.”

Alternatively, use GitHub Desktop: Click “File” > “New repository,” then follow the prompts.

Cloning a Repository

To clone a repository, copy the repository URL from GitHub.

In GitHub Desktop, click “File” > “Clone repository,” then paste the URL and choose the local path.

Making Changes and Committing

In your local repository, make changes to files as needed.

Use GitHub Desktop to view changes. Select the files to commit and add a commit message.

Click “Commit to main” (or your current branch name) to save changes locally.

Creating and Managing Branches

Branches allow you to work on different versions of your code simultaneously.

In GitHub Desktop, click “Current branch” > “New branch,” then name your branch and click “Create branch.”

Switch branches by selecting them from the “Current branch” dropdown.

Merge branches by selecting the branch you want to merge into, then click “Branch” > “Merge into current branch.”

Deleting a Branch

To delete a branch using GitHub Desktop, first switch to a different branch (you cannot delete the branch you are currently on).

Click on “Current branch” and select the branch you want to delete.

Click on “Branch” in the menu and select “Delete” from the dropdown.

Confirm the deletion when prompted.

Pushing Changes to GitHub

To push changes from your local repository to GitHub, click “Push origin” in GitHub Desktop.

Ensure you are on the correct branch and that your commits are up to date before pushing.

Pulling Changes from GitHub

To pull updates from the remote repository, click “Pull origin” in GitHub Desktop.

This ensures your local repository is synchronized with the remote repository.

Resolving Merge Conflicts

Merge conflicts occur when changes are made to the same part of a file in different branches.

GitHub Desktop will notify you of conflicts. Use the built-in conflict resolver or manually edit files to resolve conflicts.

After resolving conflicts, commit and push changes as usual.

Example Workflow

Create a new branch for a feature: “Feature/new-login-page”

Make changes and commit locally: “Added login page”

Push changes to GitHub, then create a pull request on GitHub for review.

After review, merge the pull request and delete the branch if no longer needed.
