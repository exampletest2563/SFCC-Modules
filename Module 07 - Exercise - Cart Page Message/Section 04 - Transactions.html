Transactions

Where you work with objects, whether System or Custom ones, there is one essential detail you must be aware of. You can retrieve objects as much as you want, but when you want to create, edit, or delete an object, things are different.

A transaction provides a context for performing atomic changes to persistent business objects. Before a business object can be created, changed, or deleted, a transaction must be started, and changes to the affected business objects will only be made durable when the transaction is committed.

Keep that in mind whenever you need to create, change, or delete an object instance; you will need a transaction.

Transaction.wrap(function () {
   ...
});

Transaction Types

We can have two types of transactions: Implicit and Explicit.

Implicit: This is the example we used above. You just define the code that will be inside the transaction without actually writing any begin or commit commands. If anything fails, the transaction will be rolled back automatically, and the changes will be undone.

Explicit: In this type of transaction, you are responsible for defining where the transaction begins and ends.
