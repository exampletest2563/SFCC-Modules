System and Custom Objects. SFCC B2C Site Preferences

Introduction

We are going to learn about two important SFCC features, which allow us to persist and retrieve data from Business Manager. The first one is called Objects, and we have two types: System and Custom. The second one is called Site Preferences.

Objects

How do you think Business Manager stores Products, Catalogs, or Categories? How does the API work with them?

Previously, you were working with Controllers, calling some classes from the SFCC API to get data back from your Business Manager. These objects that you are manipulating are actually stored in Business Manager as tables.

System Objects

The first type of object that can be saved into Business Manager is called a System Object. To begin with, do you know why they are called System objects? It’s simpler than you might think: they come already out-of-the-box when you receive your sandbox.

System Object Types

Let’s have a look at the system objects that we are provided out-of-the-box: Go to Administration > Development > System Objects Types

Do you recognize the items pointed to by the green arrows? They are some of the system objects we have been working on in previous chapters.

Check System Object Fields

  • Go to Administration > Development > System Objects Types
  • Click on the Product item
  • Click on the Attribute Definitions tab
  • At the bottom, click to Show All

As you can see, the Product Object has almost 100 fields! But as far as we remember, we did not see that many fields when we were editing products a few modules ago. So what is going on?

Although your objects can have many fields, to avoid cluttering the screen, we can limit which fields are displayed when we create/edit an instance of that object. If you go back to check any Product, you will notice that not even half of the fields we just saw are listed there.

So how do we decide which fields are displayed?

Check Which Fields Are Displayed in the Edit Page

  • Go to Administration > Development > System Objects Types
  • Click on the Product item
  • Click on the Attribute Grouping tab
  • Check the names in the Name column: In the Attributes column, you will see a number per row. That number represents the number of fields inside that grouping.
  • Click on the number related to Search Engine Optimization Support: You will see 4 fields listed: Page Title, Page Description, Page Keywords, and Page URL
  • Now open any product and check if these same fields are listed under the Search Engine Optimization Support set.

As you can see, this way we can customize all object pages to show the fields that we want to and hide everything else that we don’t think is necessary for that project.

One last important detail: as we mentioned at the beginning of this module, System Objects come out-of-the-box with your Business Manager. This means you can’t create new System Object Types. If you want a new type of Object, then we should see Custom Objects.
