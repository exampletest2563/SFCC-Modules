How to Access ViewModel Data

Once data is sent in our controller via setViewData() or as the second parameter in the res.render() function, it becomes accessible in our templates through the pdict variable, as we learned in previous chapters. For example, some model properties are used in the cart.isml template—take a look!

As mentioned earlier, in SFCC, ViewModels are objects created specifically for the page being rendered or for the request being responded to.

Whenever you see data being sent through the res.render or res.json functions, and the data is encapsulated in { }, that's your ViewModel.

To access this data on your page, consider it as entries in your pdict object. Let’s look at an example from the Search Controller, specifically the Refinebar endpoint.

Example: Accessing ViewModel Data in Templates

Check how the search/searchRefineBar ISML template accesses the productSearch object using the { } syntax. The objects from the controller are placed inside the pdict.

Imagine you're creating a product filter for a clothing store. The { } syntax allows you to pass only the data necessary to display product categories, sizes, and prices. This way, you're not overwhelming the page with unrelated data like customer reviews or shipping details.
