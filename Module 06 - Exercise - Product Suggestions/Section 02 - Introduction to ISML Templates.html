ISML Templates Introduction

ISML stands for Internet Store Markup Language. Files with the .isml extension define how data, tags, and page markup are transformed into HTML that is sent to the browser, with Cascading Style Sheets (CSS) used for page layout and styling.

Salesforce Commerce Cloud Digital utilizes templates to generate dynamic HTML-based web pages for responses sent back to the client.

First, let’s get started by understanding what you can do with ISML. Then, we’ll take a short tour of how ISML templates are organized on our site and finally explore how to avoid hardcoding text strings that become visible to the user.

Pdict Variable

pdict is a hashmap that allows key-object pairs to be loaded on the frontend. Some built-in pdict keys (variables) provide access to commonly used objects, such as session and request. This means that any data sent from the backend to the frontend, whether using res.render or res.setViewData, will be available in the pdict variable on the frontend.

ISML expressions provide access to data using dot notation. For example, you can access a property of the Product object in the pipeline dictionary with the expression: ${pdict.varName}.

JavaScript controllers can use alternatives to pdict keys. The table below shows some of these alternatives:

pdict Key Alternative
CurrentSession TopLevel.global.session
CurrentRequest TopLevel.global.request
CurrentCustomer TopLevel.global.customer
CurrentHttpParameterMap TopLevel.global.request.httpParameterMap
CurrentPageMetaData TopLevel.global.request.pageMetaData
CurrentForms TopLevel.global.session.forms
