ISML Tags and Expressions

ISML tags are proprietary extensions to HTML used inside ISML templates on Commerce Cloud. These tags and expressions are specific to ISML templates and cannot be used outside of them.

ISML tags always start with is, such as <isprint>, and describe how dynamic data will be embedded and formatted on the page, in conjunction with regular HTML.

ISML tags can be categorized based on their functionality, as shown in the table below:

Group Tags Purpose
HTTP-related <iscookie>, <iscontent>, <isredirect>, <isstatus> Sets cookies, defines MIME types, redirects browsers, and sets status codes.
Flow Control <isif>, <iselse>, <iselseif>, <isloop>, <isnext>, <isbreak> Evaluates conditions, specifies alternative logic, creates loops, and controls loop execution.
Variable-related <isset>, <isremove> Creates and removes variables.
Include <isinclude>, <ismodule>, <iscomponent> Includes contents of one template into another, declares custom tags, and includes output from a controller or pipeline.
Scripting <isscript> Allows execution of Commerce Cloud Digital Script inside templates.
Forms <isselect> Enhances the HTML <select> tag.
Output <isprint>, <isslot>, <iscontentasset> Formats strings for output, creates content slots, and generates content assets.
Others <iscache>, <iscomment>, <isdecorate>, <isreplace> Caches pages, adds comments, reuses templates for page layout, and replaces content inside decorator templates.
Active Data <isactivedatahead>, <isactivecontenthead>, <isobject> Collects active data, category context, and specific object impressions/views dynamically.

ISML Expressions

ISML Expressions are based on the Digital Script language, which implements the ECMAScript standard. This means that access to variables, methods, and objects in ISML expressions is similar to using JavaScript.

ISML expressions are embedded within ${â€¦} to allow the ISML processor to interpret them before executing an ISML tag or rendering the rest of the page. They can also access Digital Script classes and methods. Two packages are available implicitly in ISML, so classes do not need to be fully qualified:

  • TopLevel package: session.getCustomer()
  • dw.web package: URLUtils.url(), URLUtils.webRoot()

The TopLevel package includes a class named global, which is implied and does not need a prefix. Other classes and methods must be fully qualified, such as ${dw.system.Site.getCurrent().getName()}.

Examples of ISML expressions include:

  • ${TopLevel.global.session.getCustomer().getProfile().getLastName()} (equivalent to ${session.getCustomer().getProfile().getLastName()})
  • ${session.customer.profile.lastName} (alternative syntax)
  • ${pdict.CurrentSession.customer.profile.lastName}
  • ${pdict.CurrentCustomer.profile.lastName}
  • ${dw.system.Site.getCurrent().getName()} (alternative syntax: ${dw.system.Site.current.name})
  • ${pdict.myProduct.getLongDescription() != null} (example of complex arithmetical, boolean, and string operations)

ISML Tags

In this section, we will cover the most frequently used tags:

  • <isprint>
  • <isset>
  • <isinclude>
  • <isdecorate>
  • <isloop>
  • <isif>, <iselseif>, <iselse>

Although some ISML tags, such as <isslot>, do not require a corresponding closing tag, it is considered best practice to always use a closing tag for clarity and consistency.
