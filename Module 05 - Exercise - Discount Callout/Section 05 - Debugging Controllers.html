Debugging Controllers

One of the most important tools a developer can have is a debugger. It simplifies your development process by allowing you to pause code execution at any specific line, analyze variable values, and execute your code line by line to ensure it behaves as expected.

The Prophet extension for Visual Studio Code not only lets you sync your code with the sandbox but also provides a robust debugging feature. To use it, follow these steps:

  1. Open any controller and click on the left of any line number that has code. A red dot will appear, indicating a breakpoint. When the function is called, execution will pause at that line if the debugger is running.
  2. On the left panel, click on the bug icon.
  3. If you haven't created a debug configuration before, you will see three options: Debug with Node.js, Run with Node.js, and Create a launch.json file. Choose to create the JSON file.
  4. It will generate a file named launch.json inside a folder named .vscode. Delete its contents and replace them with the following:
{
    "version": "0.1.0",
    "configurations": [
        {
            "type": "prophet",
            "request": "launch",
            "name": "Attach to Sandbox"
        }
    ]
}

To start debugging, click the play button on the left side of the dropdown list.

For instance, if you're troubleshooting a controller method that's supposed to handle user authentication, setting a breakpoint at the beginning of the method allows you to inspect the authentication flow step by step. This can help you pinpoint where things might be going wrong, whether it's a problem with variable values or logic errors.
